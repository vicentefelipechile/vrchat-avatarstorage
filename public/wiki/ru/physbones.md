# PhysBones

<span class="badge badge-blue">ЗАВИСИМОСТЬ</span>

## Что это?
PhysBones — это набор компонентов, встроенных в VRChat SDK, который позволяет добавлять вторичное движение (физику) к объектам в аватарах и мирах. С помощью PhysBones можно добавить движение волосам, хвостам, ушам, одежде, проводам, растениям и многому другому. Правильное использование делает ваших аватаров более динамичными и реалистичными.

> [!NOTE]
> PhysBones является **официальной заменой** Dynamic Bones в VRChat. Хотя Dynamic Bones все еще работает на существующих аватарах (он преобразуется автоматически), все создатели должны использовать PhysBones для новых аватаров.

## Для чего это нужно?
- Добавить физику волосам, хвостам, ушам и одежде
- Позволить другим игрокам взаимодействовать с элементами вашего аватара (хватать, позировать)
- Создать динамическое и реалистичное вторичное движение
- Заменить компонент Cloth Unity для простых тканей

## Основные компоненты

PhysBones состоит из трех компонентов, которые работают вместе:

| Компонент | Описание |
|------------|-------------|
| **VRCPhysBone** | Основной компонент, определяющий цепочку костей, которая будет анимироваться с помощью физики |
| **VRCPhysBoneCollider** | Определяет коллайдеры, влияющие на PhysBones (голова, туловище, руки и т.д.) |
| **VRCPhysBoneRoot** | Опционально. Определяет корень движения для нескольких PhysBones (только для миров) |

## Детальная настройка

### Версии

Вы можете выбрать версию компонента VRCPhysBone непосредственно в инспекторе. По умолчанию используется последняя доступная версия.

**Версия 1.0:**
- Базовая версия компонента PhysBone

**Версия 1.1 (Squishy Bones):**
- Позволяет костям сжиматься и растягиваться
- Гравитация теперь действует как пропорция того, насколько кости будут вращаться в состоянии покоя
- Положительное Pull требуется для того, чтобы кости двигались в направлении гравитации

### Transforms

| Настройка | Описание |
|------------|-------------|
| **Root Transform** | Трансформ, где начинается компонент. Если пусто, начинается с этого GameObject |
| **Ignore Transforms** | Список трансформов, которые не должны быть затронуты компонентом |
| **Ignore Other Phys Bones** | Если включено, PhysBone игнорирует другие PhysBones в иерархии |
| **Endpoint Position** | Вектор для создания дополнительных костей в конечной точке цепочки |
| **Multi-Child Type** | Поведение корневой кости при наличии нескольких цепочек |

> [!CAUTION]
> Если вы используете одну корневую кость или корень с несколькими дочерними элементами (без внуков), вы ДОЛЖНЫ определить Endpoint Position! Это отличается от Dynamic Bones.

### Forces (Силы)

**Integration Type:**
- **Simplified**: Более стабильный, проще настраивается, менее реактивен на внешние силы
- **Advanced**: Менее стабильный, позволяет более сложные конфигурации, более реактивен на внешние силы

Доступные параметры:
- **Pull**: Сила для возврата костей в положение покоя
- **Spring** (Simplified) / **Momentum** (Advanced): Количество колебаний при попытке достичь положения покоя
- **Stiffness** (только Advanced): Степень усилия оставаться в положении покоя
- **Gravity**: Количество применяемой гравитации. Положительное значение тянет вниз, отрицательное — вверх
- **Gravity Falloff**: Контролирует, сколько гравитации удаляется в положении покоя (1.0 = нет гравитации в покое)

> [!TIP]
> Если ваши волосы смоделированы в положении, которое вы хотите, когда стоите нормально, используйте Gravity Falloff = 1.0. Так гравитация не будет влиять на вас, когда вы стоите неподвижно.

### Limits (Ограничения)

Ограничения позволяют ограничить, насколько может двигаться цепочка PhysBones. Они очень полезны для предотвращения прохождения волос через голову и **гораздо более производительны**, чем коллайдеры.

| Тип | Описание |
|------|-------------|
| **None** | Без ограничений |
| **Angle** | Ограничено максимальным углом от оси. Визуализируется как конус |
| **Hinge** | Ограничено вдоль плоскости. Похоже на кусок пиццы |
| **Polar** | Объединяет Hinge с Yaw. Более сложный, использовать с осторожностью |

> [!WARNING]
> Не злоупотребляйте полярными ограничениями. Использование более 64 может вызвать проблемы с производительностью.

### Collision (Коллизии)

| Настройка | Описание |
|------------|-------------|
| **Radius** | Радиус коллизии вокруг каждой кости (в метрах) |
| **Allow Collision** | Позволяет коллизию с глобальными коллайдерами (руки других игроков, коллайдеры мира) |
| **Colliders** | Список конкретных коллайдеров, с которыми этот PhysBone сталкивается |

**Опции Allow Collision:**
- **True**: Сталкивается с глобальными коллайдерами
- **False**: Сталкивается только с перечисленными коллайдерами
- **Other**: Расширенные опции для фильтрации по типу (аватар, мир, предмет)

### Stretch & Squish (только v1.1)

| Настройка | Описание |
|------------|-------------|
| **Stretch Motion** | Количество движения, влияющее на растяжение/сжатие костей |
| **Max Stretch** | Максимально допустимое растяжение (кратное исходной длине) |
| **Max Squish** | Максимально допустимое сжатие (кратное исходной длине) |

### Grab & Pose (Хватать и Позировать)

| Настройка | Описание |
|------------|-------------|
| **Allow Grabbing** | Позволяет игрокам хватать кости |
| **Allow Posing** | Позволяет игрокам позировать после хватания |
| **Grab Movement** | Контролирует, как кости двигаются при хватании (0 = использует pull/spring, 1 = мгновенно) |
| **Snap To Hand** | Кость автоматически подстраивается под руку, которая её хватает |

## Практические примеры использования

### Пример 1: Длинные волосы

1. Выберите корневую кость волос (обычно на шее или голове)
2. Добавьте компонент **VRCPhysBone**
3. Настройте:
   - **Root Transform**: Корневая кость волос
   - **Ignore Transforms**: Глаза и любые кости, которые не должны двигаться
   - **Multi-Child Type**: Ignore (чтобы все кости волос обрабатывались одним компонентом)
   - **Pull**: 0.3 - 0.5
   - **Gravity**: 0.5 - 1.0
   - **Gravity Falloff**: 0.5 - 0.8 (настройте в зависимости от того, как вы хотите, чтобы они падали в покое)
   - **Radius**: 0.05 - 0.1
4. Добавьте **Limits** типа Angle, чтобы волосы не проходили сквозь голову

> [!TIP]
> Для очень длинных волос рассмотрите возможность разделения на несколько компонентов PhysBone (по одному для каждой секции) для лучшей производительности.

### Пример 2: Хвост животного

1. Выберите базовую кость хвоста
2. Добавьте компонент **VRCPhysBone**
3. Настройте:
   - **Root Transform**: Базовая кость хвоста
   - **Integration Type**: Advanced
   - **Pull**: 0.2 - 0.4
   - **Spring/Momentum**: 0.5 - 0.7
   - **Stiffness**: 0.1 - 0.3
   - **Gravity**: 0.3 - 0.6
4. Используйте ограничения **Hinge** для ограничения бокового движения

### Пример 3: Юбка или плащ

1. Убедитесь, что одежда имеет свою собственную арматуру, отдельную от аватара
2. Выберите корневую кость юбки/плаща
3. Добавьте компонент **VRCPhysBone**
4. Настройте:
   - **Pull**: 0.1 - 0.3 (мягче для тканей)
   - **Gravity**: 0.8 - 1.0
   - **Gravity Falloff**: 0.3 - 0.5
   - **Radius**: 0.05
5. Добавьте **VRCPhysBoneCollider** к туловищу аватара
6. В компоненте PhysBone, в **Colliders**, добавьте коллайдер туловища

> [!NOTE]
> Для очень длинных юбок или полных плащей рассмотрите использование компонента Cloth Unity вместо PhysBones, так как он оптимизирован для этого типа ткани.

## Dynamic Bones против PhysBones

VRChat автоматически преобразует компоненты Dynamic Bones в PhysBones при загрузке аватара. Однако это преобразование не идеально.

**Основные отличия:**
- Dynamic Bones использует режим Advanced по умолчанию при преобразовании
- Некоторые настройки Dynamic Bones не имеют эквивалента в PhysBones
- Автоматическое преобразование использует "Ignore" для Multi-Child Type

**Ручное преобразование:**
Вы можете вручную преобразовать свои аватары, используя VRChat SDK → Utilities → Convert DynamicBones to PhysBones.

> [!WARNING]
> Сделайте резервную копию аватара перед преобразованием, так как процесс необратим.

## Ограничения и производительность

| Платформа | Ограничение |
|------------|-----------|
| **ПК** | ~256 трансформов на компонент |
| **Meta Quest** | Более низкий ограничение (см. документацию Performance Ranking) |

**Советы по оптимизации:**
- Не используйте более 256 трансформов на компонент PhysBone
- Если у вас более 128 трансформов, рассмотрите возможность разделения на несколько компонентов
- Используйте **Limits** вместо коллайдеров, когда возможно
- Не используйте гуманоидные кости (Hip, Spine, Chest, Neck, Head) в качестве корней PhysBones

> [!IMPORTANT]
> PhysBones имеет жесткое ограничение на Meta Quest. Обратитесь к ограничениям "Very Poor" в системе Performance Ranking.

## Распространенные ошибки

### PhysBone не двигается
- Проверьте, что Root Transform правильно назначен
- Убедитесь, что он не установлен в "Ignore" в Multi-Child Type
- Проверьте, что значение Pull не равно 0

### PhysBone проходит сквозь тело
- Добавьте ограничения (Limits) к компоненту
- Добавьте коллайдеры к аватару и настройте их в PhysBone
- Увеличьте значение Pull

### Кости не достигают положения покоя
- Увеличьте значение Pull
- Настройте Spring/Momentum в зависимости от типа интеграции

### Кости проходят сквозь тело
- Добавьте VRCPhysBoneCollider к аватару
- Настройте коллайдер в списке Colliders PhysBone
- Проверьте, что Radius соответствует
